## Plugin type

Paper Entry is a **quiz report subplugin** of type `quiz` (not `mod`). Its
component name is `quiz_paperentry` and it lives at:

```
mod/quiz/report/paperentry/
```

Moodle discovers it automatically via the `mod/quiz/report/` scan in the quiz
module bootstrap.

## Class responsibilities

### `export.php` вЂ” `quiz_paperentry\export`

- `get_questions(int $quizid): array` вЂ” returns all multichoice/truefalse questions
  in quiz slot order; appends `_slot{n}` suffix to duplicate names.
- `get_enrolled_students(int $courseid): array` вЂ” returns enrolled users with
  student role (excludes teachers and managers).
- `generate_csv(int $quizid, array $extrafields): string` вЂ” builds the grading sheet
  string.
- `csv_safe(string $value): string` вЂ” prefixes formula-injection characters.

### `import.php` вЂ” `quiz_paperentry\import`

- `process(string $csvpath, int $quizid): bool` вЂ” two-pass import entry point.
  Returns `true` on full success, `false` if any validation error occurred.
- `validate_row(array $row, int $rownum): ?array` вЂ” validates one CSV row; returns
  `['userid' => int, 'answers' => array]` or `null` (skip row) or appends to
  `$this->errors`.
- `create_attempt(int $userid, array $answers): void` вЂ” commits a single student's
  attempt using the Moodle quiz engine API.
- `get_errors(): array`, `get_warnings(): array`, `get_imported(): int` вЂ” result
  accessors used by the rendering layer.

### `settings_manager.php` вЂ” `quiz_paperentry\settings_manager`

- Quiz settings CRUD (`get_settings`, `save_settings`).
- Grader management (`add_grader`, `remove_grader`, `get_graders`).
- File API bridge (`save_submission`, `get_submission_file`,
  `get_all_submissions_csv`).
- Role management (`get_or_create_grader_role`).

### `privacy/provider.php`

Implements:
- `core_privacy\local\metadata\provider`
- `core_privacy\local\request\plugin\provider`
- `core_privacy\local\request\core_userlist_provider`

## AMD modules

All JavaScript is loaded via `$PAGE->requires->js_call_amd()` вЂ” never inline.
The three AMD modules are in `amd/src/`:

| Module | `init()` signature | Purpose |
|---|---|---|
| `quiz_paperentry/mapping_widget` | `(widgetid, fromlabel, tolabel, injectintoid)` | Value-substitution tag-picker |
| `quiz_paperentry/edit_answer` | `(qdata, currentanswers, noattemptstr, currentstr)` | In-place answer editor |
| `quiz_paperentry/export_settings` | `(widgetid, availablefields, selectedfields)` | Export-column tag-picker |

AMD build output goes to `amd/build/` (generated by `grunt amd`).

## File API storage

Grader CSV submissions are stored in the Moodle File API under:

| Key | Value |
|---|---|
| `component` | `quiz_paperentry` |
| `filearea` | `submission` |
| `contextid` | Module context id of the quiz |
| `itemid` | Moodle user id of the grader |
| `filepath` | `/` |
| `filename` | `submission.csv` |

This means files are:
- Governed by Moodle's file permissions system.
- Included in Moodle's GDPR data export/deletion for the grader.
- Backed up if the Moodle backup includes the module context files (note:
  quiz report plugins are not currently called during backup вЂ” see
  [Backup & Restore](backup-restore/)).
