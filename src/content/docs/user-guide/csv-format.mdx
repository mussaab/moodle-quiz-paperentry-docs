## File requirements

| Property | Requirement |
|---|---|
| Encoding | **UTF-8** (BOM is tolerated) |
| Delimiter | Comma (`,`) |
| Line ending | LF or CRLF |
| File extension | `.csv` |
| Header row | **Required** (first row) |
| Maximum rows | No hard limit — limited by PHP `memory_limit` and `max_execution_time` |

---

## Header row

The first row must contain column headers. The plugin uses header values to match
columns to questions and profile fields.

### Question columns

Each question is identified by its **name** (the "Question name" field in the Moodle
question editor). Column headers must match question names exactly
(case-insensitive, leading/trailing whitespace is ignored).

If two questions in the quiz share the same name, the export appends a slot suffix
(`_slot2`, `_slot3`, …) to disambiguate:

```
Q1,Q2,Q2_slot3,Q3
```

### Profile columns

Extra columns (email, ID number, custom fields) use the Moodle profile field
shortname:

| Column header | Meaning |
|---|---|
| `email` | Student's Moodle account email |
| `idnumber` | Institutional ID number |
| `firstname` | First name |
| `lastname` | Last name |
| `profile_field_class` | Custom profile field with shortname `class` |

The plugin uses the first profile column it recognises to identify the student row.
Unknown column headers are silently ignored.

## Answer values

### Multichoice questions

Enter the answer text exactly as it appears in the question options. The match is
**case-insensitive** and **strips HTML tags** and leading/trailing whitespace:

| Question option | Valid CSV values |
|---|---|
| `Paris` | `paris`, `PARIS`, `Paris` |
| `<b>True</b>` | `true`, `TRUE`, `True` |

<Aside type="tip">
  Use [Value Substitutions](../../configuration/value-substitutions/) to map short
  notations (e.g. `a`, `b`, `c` or `1`, `2`, `3`) to full answer texts.
</Aside>

### True/False questions

True/false questions accept the localised true/false string (e.g. `True` / `False`
in English). Value substitutions are the recommended way to let graders enter `T`/`F`.

### Empty cells

Leave a cell empty to record "no response" for that question. The student will
receive 0 marks for the question.

## Example

A quiz with questions "Capital" (multichoice: Paris / London / Berlin) and
"Island?" (true/false), exported with an `idnumber` column:

```csv
idnumber,Capital,Island?
S001,Paris,True
S002,London,False
S003,Berlin,True
S004,,
```

Row 4: student `S004` left both questions unanswered.

## Validation rules

The import performs a **two-pass validation**:

1. **Pass 1** — every row is checked before anything is written to the database.
   - Unknown student identifier → row is **skipped** (warning, not error).
   - Unrecognised answer value → **error**; the entire import is aborted.
   - Missing required profile column → **error**.
2. **Pass 2** — if zero errors, all validated rows are committed to the database.

This means you either get a complete successful import or nothing is written at all —
there is no partial import state.

---

## Formula-injection protection

All values written to exported CSVs are passed through `csv_safe()`, which prefixes
any cell that starts with `=`, `+`, `-`, or `@` with a single quote. This prevents
Excel and LibreOffice from interpreting cell values as formulas.

<Aside type="caution">
  This protection applies to the **export** direction only. Graders' uploaded files
  are read as raw text and never executed, so formula injection from uploads is not
  a concern.
</Aside>
